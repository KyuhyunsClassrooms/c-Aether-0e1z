## 🐛 발견한 오류 #1

### 1) 어떤 문제가 발생했나요?
(예: 프로그램을 실행하면 이상한 값이 나왔어요 / 컴파일이 안 돼요)

```
기차 목록을 출력할 때, 항상 “등록된 기차 정보가 없습니다.” 라고 출력되었어요.

### 2) 오류가 발생한 코드
```

```c
// if (count = 0) {
    printf("등록된 기차 정보가 없습니다.\n");
    return;
}
```

### 3) 왜 오류가 생겼나요?
(예: 세미콜론(;)을 빼먹었어요 / 변수 이름을 잘못 썼어요)

```
== 를 써야 하는데 실수로 대입 연산자(=) 를 사용해서
조건문이 항상 참(0 → false, 그러나 대입되면서 조건문은 false가 아닌 값으로 동작) 으로 처리됨.
```

### 4) 어떻게 고쳤나요?
```c
// if (count == 0) {
    printf("등록된 기차 정보가 없습니다.\n");
    return;
```

### 5) 고친 후 결과
- [o] ✅ 잘 작동해요!
- [ ] ❌ 아직 문제가 있어요

## 🐛 발견한 오류 #2

### 1) 어떤 문제가 발생했나요?
```
기차 번호 입력 후 문자열 입력을 받을 때
출발역이 스킵되고 바로 도착역을 입력받는 문제가 발생했어요.
```

### 2) 오류가 발생한 코드
```c
// scanf("%d", &trains[count].trainNumber);
scanf("%s", trains[count].departure);
```

### 3) 왜 오류가 생겼나요?
```
scanf("%d") 뒤에 엔터(개행문자)가 입력 버퍼에 남아 있어
바로 다음 scanf("%s")가 그것을 입력으로 받아버림.
```

### 4) 어떻게 고쳤나요?
```c
// scanf("%d", &trains[count].trainNumber);
getchar();  // 버퍼 비우기
scanf("%s", trains[count].departure);
```

### 5) 고친 후 결과
- [o] ✅ 잘 작동해요!
- [ ] ❌ 아직 문제가 있어요

### 1) 어떤 문제가 발생했나요?
```

```

### 2) 오류가 발생한 코드
```c
// if (trains[i].trainNumber = searchNo) {
```

### 3) 왜 오류가 생겼나요?
```
비교 연산(==)이 아니라 대입 연산자(=) 을 사용하여
모든 trainNumber가 searchNo로 바뀌어 버림.
```

### 4) 어떻게 고쳤나요?
```c
// if (trains[i].trainNumber == searchNo) {
```

## 💭 디버깅하면서 배운 점

이번 프로젝트에서 오류를 고치면서 새롭게 알게 된 점이나 느낀 점을 자유롭게 적어보세요.

```
- 비교 연산자(==)와 대입 연산자(=)는 완전히 다르다는 것을 확실히 배웠어요.
- scanf("%d") 뒤에는 버퍼가 남아서 문제가 생길 수 있다는 것을 알게 되었어요.
- 에러가 나면 무서워하지 말고, 출력문을 이용해 어디서 문제가 생기는지 찾는 게 중요하다는 걸 느꼈어요.
- 디버깅 과정이 처음엔 어렵지만, 하나씩 고쳐 나갈수록 프로그램이 완성되는 느낌이 흥미로웠어요.
```
